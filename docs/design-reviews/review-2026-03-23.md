## Gut Reaction

The dashboard communicates **control surface**. Dark, dense, information-first. The tinted card section headers (amber approvals, emerald standups, indigo mission) are the best design decision — they add warmth and wayfinding without decoration. The 11px uppercase mono micro-labels are the signature move. The industrial feel is real.

**But** there's a civil war in the CSS. Half the codebase follows DESIGN.md (`rounded-[2px]`, `rounded-[6px]`, `p-[20px]`, `border-border`). The other half uses competing patterns (`rounded-sm`, `rounded-md`, `p-3`/`p-4`/`p-5`, `border-zinc-800`, `bg-[#121214]`). It's the same visual output in many cases — but two notation systems means two sources of truth, and drift is inevitable.

**First 3 things the eye is drawn to:** the tinted section headers, the 30px entity names, the mono micro-labels.

**One-word verdict:** Bifurcated.

## Top 5 Design Improvements

### 1. Unify the token system — kill all hardcoded hex

**What:** 77 instances of `bg-[#121214]` and `bg-[#09090b]` exist alongside their semantic equivalents `bg-card` and `bg-background`. DESIGN.md rule #9 explicitly bans this.

**Why it matters:** Two notation systems = eventual drift. When you change the card background color, you'd need to update both `bg-card` in Tailwind config AND find-replace 47 hardcoded hex values. One will get missed.

**Severity:** High | **Category:** Color

**Fix:** Global find-replace: `bg-[#121214]` → `bg-card`, `bg-[#09090b]` → `bg-background`. Same visual output, single source of truth.

---

### 2. Unify border-color notation

**What:** Exactly 111 instances of `border-zinc-800` and 111 instances of `border-border`. They resolve to the same color. Two systems, perfectly split.

**Why it matters:** Same drift risk as #1. `border-border` is the semantic token. `border-zinc-800` is the hardcoded equivalent.

**Severity:** High | **Category:** Color

**Fix:** Global find-replace: `border-zinc-800` → `border-border`. 111 changes, zero visual difference.

---

### 3. Standardize card padding to `p-[20px]`

**What:** DESIGN.md specifies `p-[20px]` for card internals. Only 18 instances follow this. The codebase has 78× `p-3` (12px), 44× `p-5` (20px), 41× `p-4` (16px). Note: `p-5` IS 20px — same value as `p-[20px]`, different notation.

**Why it matters:** Inconsistent density. Some cards feel tight (`p-3`), some breathe (`p-5`). The dashboard should have uniform information density.

**Severity:** High | **Category:** Spacing

**Fix:** Audit each `p-3`/`p-4` on card bodies — most should be `p-[20px]` or `p-5`. The two notations (`p-5` vs `p-[20px]`) should also converge to one.

---

### 4. Consolidate border-radius notations

**What:** Cards use both `rounded-[2px]` (62×) and `rounded-sm` (41×). Interactive elements use both `rounded-[6px]` (36×) and `rounded-md` (35×). These are the same values expressed differently.

**Why it matters:** Semantic clarity. `rounded-[2px]` communicates "this is a deliberate design token." `rounded-sm` communicates "Tailwind default." The intent is different even if the output is the same.

**Severity:** Medium | **Category:** Layout

**Fix:** Pick one notation per role and enforce it. DESIGN.md says `rounded-[2px]` and `rounded-[6px]` — standardize to those.

---

### 5. Fix focus ring inconsistency

**What:** DESIGN.md specifies `focus:ring-[3px] focus:ring-ring/50`. Code has three competing patterns: `focus:ring-[3px]` (26×), `focus:ring-1` (17×), and `focus:ring-2` (5×). Zero `focus-visible:` variants outside of `index.css`.

**Why it matters:** Keyboard navigation has inconsistent visual feedback. Some inputs get a 3px ring, others get 1px, others get 2px. And mouse clicks trigger focus rings unnecessarily because `focus-visible:` isn't used.

**Severity:** Medium | **Category:** Interaction States

**Fix:** Replace all `focus:ring-*` with `focus-visible:ring-[3px] focus-visible:ring-ring/50`. Standardize on the DESIGN.md pattern.

## Inferred Design System (what actually shipped)

### Fonts
- **2 families:** System sans-serif + System mono (SF Mono)
- **Heading scale:** 30px (entity) → 16px (page title) → 15px (section header, buttons, breadcrumbs) → 14px (body) → 12px (small) → 11px (micro-label) → 10px (tiny badges)
- **Ratio h1:h2 = 30:16 = 1.875:1** — good drama
- **Signature move:** 11px uppercase mono tracking-[0.15em] — 93 instances, used consistently

### Colors
- **Achromatic base:** oklch grays (0.145 background → 0.985 foreground)
- **8 semantic accent colors:** emerald (success), indigo (project/info), amber (warning/approval), red (error), cyan (running/experiment), violet (proposed/issue), blue (completed/comments), zinc (neutral)
- **No brand color, no decorative color** — correct for ops dashboard
- **Tinted card headers** — subtle `bg-{color}-500/[0.02]` wash, strongest visual identity element

### Spacing
- **Base unit:** 4px (Tailwind)
- **Card padding:** chaotic — `p-[20px]`, `p-5`, `p-4`, `p-3` all in use
- **Section spacing:** `space-y-6` (24px) — consistent
- **Metric card gaps:** `gap-2` (8px) — consistent
- **Content area:** `p-6 md:p-8` on list pages

### Border Radius
- **Cards:** `rounded-[2px]` OR `rounded-sm` (2px) — two notations, same value
- **Interactive:** `rounded-[6px]` OR `rounded-md` (6px) — two notations, same value
- **Pills/badges:** `rounded-full` — consistent (102 instances)
- **No `rounded-lg`** — correctly absent

### Borders & Shadows
- **Card borders:** `border border-zinc-800` OR `border border-border` — split 50/50
- **Shadows:** `shadow-sm` only (73 instances) — consistent
- **Row dividers:** `border-b border-zinc-800/50` or `border-b border-border/50`

## Recommendations

1. **Your DESIGN.md is drifting from what shipped.** The design system is excellent on paper. The code follows the spirit but uses competing notations for the same values. The fix is mechanical — global find-replace on 6 patterns would eliminate ~250 inconsistencies with zero visual change.

2. **Run `design-qa` to fix these findings.** The top 3 issues (hardcoded hex, border tokens, card padding) are pure find-replace — no judgment calls needed. Border-radius and focus ring unification are also mechanical.

3. **Priority order for fixes:**
   - `bg-[#121214]` → `bg-card` (47 changes)
   - `bg-[#09090b]` → `bg-background` (23 changes)
   - `border-zinc-800` → `border-border` (111 changes)
   - `rounded-sm` → `rounded-[2px]` on cards (41 changes)
   - `rounded-md` → `rounded-[6px]` on interactive elements (35 changes)
   - `focus:ring-1` / `focus:ring-2` → `focus:ring-[3px] focus:ring-ring/50` (22 changes)
   - Card padding audit: `p-3`/`p-4` → `p-[20px]` where appropriate
